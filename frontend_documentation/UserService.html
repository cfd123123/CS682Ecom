Class: UserService

UserService

UserService manages all backend communication related to Users. It also contains a small number of functions related to navigation and shopping cart.

new UserService()

Methods

addToCart(productID, quantity)

Adds the given quantity of the given productID to the current user's cart, then calls updateCurrentUser() Only users with the 'CUSTOMER' role have access to the backend mapping associated with this request. WARNING: This function is incomplete, as it does not allow a user to add items to the cart unless that user is logged in.
Parameters:
Name Type Description
productID the product to be added to the cart
quantity the quantity of productID to be added to the cart

getAdminContent()

Sends a GET request to the backend for the currently logged in user's admin content. The server's reply with the data is returned back to the calling function. Only Admins have access to the backend mapping associated with this request. NOTE: This feature is incomplete and currently receives a text string that reads "Admin Content".
Returns:
the current user's admin content
Type
Promise.<AxiosResponse.<T>>

getCurrentUser()

Returns the current value of "user" in localStorage, or the "None" user if the key does not exist.
Returns:
Type
currentUser

getCurrentUsername()

Helper function that gets the currently logged in user's username. If the current user is not logged in, "None" is returned.
Returns:
the currently logged in user's username, or "None" if not logged in.
Type
string

getEmployeeContent()

Sends a GET request to the backend for the currently logged in user's employee content. The server's reply with the data is returned back to the calling function. Only Employees and Admins have access to the backend mapping associated with this request. NOTE: This feature is incomplete and currently receives a text string that reads "Employee Content".
Returns:
the current user's employee content
Type
Promise.<AxiosResponse.<T>>

getMyAccount()

Sends a GET request to the backend for the currently logged in user's account data. The server's reply with account data is returned back to the calling function. All users with an account and that are currently logged in have access to the backend mapping associated with this request.
Returns:
the current user's account data
Type
Promise.<AxiosResponse.<T>>

getMyStuff()

Sends a GET request to the backend for the currently logged in user's personalized content. The server's reply with profile data is returned back to the calling function. Only users with the 'CUSTOMER' role have access to the backend mapping associated with this request. NOTE: This function is incomplete and currently receives the same data as the getMyAccount() function.
Returns:
the current user's profile data
Type
Promise.<AxiosResponse.<T>>

placeOrder(orderID)

Completes the checkout process by sending a POST request to the backend with the current user's username and the PreOrder ID for this order. Only users with the 'CUSTOMER' role have access to the backend mapping associated with this request.
Parameters:
Name Type Description
orderID the PreOrder Id for this order
Returns:
the OrderResponse from the backend, containing the Order details
Type
Promise.<AxiosResponse.<T>>

proceedToCheckout(cart, total)

Begins the checkout process by sending a POST request to the backend with username and cart details. The backend response with a PreOrder is returned back to the calling function. Only users with the 'CUSTOMER' role have access to the backend mapping associated with this request.
Parameters:
Name Type Description
cart the cart details for current user's order
total the product subtotal of the cart
Returns:
the PreOrder details for the products in the user's cart
Type
Promise.<AxiosResponse.<T>>

updateCurrentUser(newUser)

Updates the "user" localStorage item and "currentUser" global variable to include newUser as the new currentUser. accessToken and tokenType are taken from existing values.
Parameters:
Name Type Description
newUser the new data to replace existing currentUser.
